Motion

Motion in Strand is precise and restrained. Small translate distances, no bounce or elastic easing, and every animation respects prefers-reduced-motion.

Easing Functions

Four easing curves cover all animation needs. Watch the dot traverse each track to see the curve in action.

--strand-ease-out-expo
cubic-bezier(0.16, 1, 0.3, 1)
Enter/open: snappy start, gentle settle
--strand-ease-out-quart
cubic-bezier(0.25, 1, 0.5, 1)
General smooth transitions
--strand-ease-in-out-sine
cubic-bezier(0.37, 0, 0.63, 1)
Gentle, scientific, balanced
--strand-ease-in-expo
cubic-bezier(0.7, 0, 0.84, 0)
Exit/close: slow start, fast finish

Duration Scale

Five duration stops calibrated to match the perceived weight of different UI changes.

--strand-duration-instant / 75ms
--strand-duration-fast / 150ms
--strand-duration-normal / 250ms
--strand-duration-slow / 400ms
--strand-duration-glacial / 700ms

Usage

.card {
  transition:
    transform var(--strand-duration-fast) var(--strand-ease-out-expo),
    box-shadow var(--strand-duration-fast) ease;
}

.card:hover {
  transform: translateY(-2px);
  box-shadow: var(--strand-elevation-2);
}

Animation Principles

What signals precision

  • Small translate distances (16-32px)
  • No bounce or elastic easing
  • Short stagger intervals (60-100ms)
  • Opacity always animates (things fade, never pop)
  • Transform-only animations (GPU compositing, 60fps)

Never

  • Re-animate on every viewport entry/exit
  • Horizontal scroll hijacking
  • Override native scroll
  • Animations longer than 0.6s that block content
  • More than 3-4 elements animating simultaneously
  • Parallax on text

Reduced Motion

All animations must respect prefers-reduced-motion: reduce. Durations drop to near-zero and scroll behavior becomes auto. This is handled automatically by the Strand reset.
