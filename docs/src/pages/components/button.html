Components

Button

Buttons trigger actions. Four variants and three sizes cover all interaction contexts. Every button meets the 44px minimum touch target.

Variants

Primary

For the main action on a page or in a form. One primary button per context.

<button class="strand-btn strand-btn--primary strand-btn--md">
  <span class="strand-btn__content">Primary</span>
</button>

Secondary

For supporting actions. Bordered style with elevated background.

<button class="strand-btn strand-btn--secondary strand-btn--md">
  <span class="strand-btn__content">Secondary</span>
</button>

Ghost

For tertiary actions or inline controls. Transparent background, blue text.

<button class="strand-btn strand-btn--ghost strand-btn--md">
  <span class="strand-btn__content">Ghost</span>
</button>

Danger

For destructive actions (delete, remove, cancel). Red background draws attention to irreversible choices.

<button class="strand-btn strand-btn--danger strand-btn--md">
  <span class="strand-btn__content">Danger</span>
</button>

<button class="strand-btn strand-btn--primary strand-btn--md">...</button>

Props

Prop Type Default Description
variant "primary" | "secondary" | "ghost" | "danger" "primary" Visual style variant
size "sm" | "md" | "lg" "md" Button size (height: 32/40/48px)
loading boolean false Show spinner and disable interaction
iconOnly boolean false Square button for icon-only use
fullWidth boolean false Stretch to full container width
disabled boolean false Disable the button

CSS Classes

Class Purpose
.strand-btn Base button styles
.strand-btn--primary Primary variant (blue background)
.strand-btn--secondary Secondary variant (bordered)
.strand-btn--ghost Ghost variant (transparent)
.strand-btn--danger Danger variant (red background)
.strand-btn--sm Small size (32px height)
.strand-btn--md Medium size (40px height)
.strand-btn--lg Large size (48px height)
.strand-btn--loading Loading state (disables pointer events)
.strand-btn--icon-only Square button for icons
.strand-btn--full-width Full container width

Accessibility

  • All buttons use native <button> elements for keyboard and screen reader support.
  • Disabled buttons set both disabled and aria-disabled="true".
  • Loading buttons set aria-busy="true" and the spinner is hidden from screen readers with aria-hidden="true".
  • Focus indicators use a 2px blue ring with 2px offset (:focus-visible).
  • All sizes meet the 44px minimum touch target (small size is 32px height but should be used in desktop contexts only).
  • Color contrast on all variants meets WCAG 2.2 AA for interactive elements.
